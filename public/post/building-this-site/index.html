How I built this website

Last updated on 14 Mar, 2020 5 min read Personal
Long lost code for WAD

For those who have no time, I summarise as:

  • Reading documentation from blogdown,
  • A fair amount of trial and error, and,
  • Plenty of frustration (aka debugging).
This post is not yet complete and perhaps never will be. I believe one of my key weaknesses (I have many) is premature optimisation and so I have posted this as-is. To prove this point - consider the last modified date and look at the Github repo for a history of this page.

The inspiration for this site

I don’t recall the motivation to build this website other than following on from a quote (potentially misquoted as from memory from over five years ago) from the very first professional interview I had with Mike Froom when I was applying for graduate employment at KPMG Birmingham:

“You should always challenge yourself by trying something new every six months.”

I didn’t think much of this at the time but having spent nearly ten years in employment I do think this is good advice. Whilst I no longer work for KPMG, I do reflect fondly on the experience as an excellent well-rounded UK graduate opportunity.

So how was this website built and how can you build your own website, for free, hosted on GitHub? What follows is an insight into how I came to build this website.

Bookdown - an R package that facilitates writing books and long-form articles/reports with R Markdown.

Bookdown is an excellent resource for anyone interested in R. Most resources on here are fully published books that are available, for free, in digital format. Specifically, my interest was piqued by the book hosted on here for the R package blogdown. Please consider supporting the authors here. As an aside, I spent some time delving into one of the author’s personal struggles he faced when transitioning from academia to industry which I found quite insightful. Here is the link to his blog post on the subject.

The book advocates using Hugo to build your statically generated website. I’ve only tried to build a website twice before this:

  1. Dabbling with Dreamweaver as a kid and not getting very far,
  2. Building a lottery based website, Warwick Accumulator Draw “WAD” as a university module, which was a lot of fun but a lot of work too. This was built back in early 2011 on Microsoft’s .NET framework.

As part of my work at Hiscox I occasionally work in R. My preferred IDE for such work is R Studio. R is fantastic for data - loading, transformation and analysis and R Studio is the best IDE for working in R in my opinion. However, I personally find R Studio a little cumbersome for web development despite the existence of the excellent blogdown package. Due to this I backtracked somewhat to read up on Hugo documentation. Here I discovered the Academic theme, which as of January 2020, largely powers the user experience on this website.

How this site was built

So how specifically was this website built? The truth is that it took a fair amount of trial and error and reading plenty of documentation. I am certainly no software developer so I hope this can give anyone the confidence that they can also build their own 100% free website.

Here are the tools I’ve used to build this website:

  1. Git - download and install Git for free (subject to the GNU General Public License version 2.0),
  2. Hugo - pick your own operating system’s binary release for free (subject to the Apache License version 2.0),
  3. GitHub - set up your own GitHub account for free,
  4. Academic Theme - fork the Academic Theme for free (subject to the MIT license),
  5. Visual Studio Code - download and install Microsoft Corporation’s Visual Studio Code for free (subject to the MIT license),

The specific deployment to host on GitHub was based on documentation from Academic.

Here is my specific setup on my Windows OS:

  • Use OneDrive for the Hugo binary in: /Hugo/bin,
  • Use OneDrive to host my local repo across my PC and my laptop within /Hugo/Sites,
  • This local repo mirrors my privately hosted GitHub (“remote”) repo that contains the source code to build the website,
  • Add content in Visual Studio Code, using Git commands to add, commit and push to remote any changes,
  • Using the Git bash command “hugo”, I re-build my website to populate the /public sub-directory,
  • Clone my built website in a local repo within OneDrive/Hugo/Sites/“website-name” so that I can Git commit changes on each re-build,
  • Copy-paste the contents of the /public sub-directory, add to Git, commit and push to remote,
  • Using the Git bash command “rm -rf public” within the local repo, I delete the /public sub-directory,
  • Repeat the above steps for publishing new content.

This deployment workflow was based on documentation within Academic.

I’ve only used Git for a few months, have barely scratched the surface of its power, but already consider it the most important tool anyone should use for any development. Perhaps it deserves its own post in the future.

Even newer than this is my use of Visual Studio Code which I consider a fantastic text editor. It’s fast, free, includes Git integration, has syntax highlighting and the UI just makes sense (e.g. the explorer side-panel for your directory tree).
